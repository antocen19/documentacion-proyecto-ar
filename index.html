Documentación de la solución de visualización inmersiva.

Iniciar Experiencia de RA

1. Análisis del Problema y Justificación del Sistema

El problema identificado es la limitación de la visualización 2D en la enseñanza del hardware (CPU, GPU, RAM), que no permite entender correctamente su escala y ubicación espacial. La solución es un sistema WebAR (Realidad Aumentada Web). Elegimos esta tecnología por su accesibilidad (no requiere instalar apps) y su capacidad para ofrecer una visualización inmersiva 3D directamente sobre un marcador físico. Este enfoque transforma un concepto abstracto en una experiencia de aprendizaje tangible y eficiente.

2. Diseño y Componentes de la Solución

El diseño se basa en el seguimiento de marcadores (marker tracking) para la activación de modelos 3D. Cada marcador físico está asociado a un componente de PC. Puedes escanear las siguientes imágenes desde otra pantalla o dispositivo móvil:

CPU (Unidad Central de Procesamiento)

Activa el modelo CPU. Es el "cerebro" encargado de ejecutar todas las instrucciones y cálculos del sistema. Este marcador representa la complejidad del componente central.

GPU (Unidad de Procesamiento Gráfico)

Activa el modelo GPU. Especializado en el renderizado de imágenes y tareas de cálculo paralelo. El diseño del marcador fue elegido por su alto contraste y fiabilidad.

RAM (Memoria de Acceso Aleatorio)

Activa el modelo RAM. Memoria de alta velocidad utilizada para almacenar datos temporales. Este patrón simple garantiza una detección rápida para la visualización inmediata.

3. Arquitectura Técnica y Stack de Desarrollo

  1. El proyecto utiliza un stack de código abierto para su desarrollo y despliegue.
  2. Librería Principal: MindAR: Es la librería de JavaScript ligera y de alto rendimiento que se encarga del reconocimiento del marcador y el cálculo de la posición de la cámara (WebXR).
  3. Contenido 3D: Modelos optimizados en formato GLTF/GLB.
  4. Plataforma de Despliegue: GitHub Pages: Se usa para el hosting estático, lo cual es crucial porque garantiza el protocolo HTTPS, un requisito fundamental para que los navegadores permitan el acceso a la cámara.
  5. Herramientas de Edición: El código se gestionó y editó usando Visual Studio Code.

4. Guía de Uso y Acceso

Sigue estos pasos para interactuar con la RA:

  1. Abre esta página web en el navegador de tu móvil.
  2. Presiona el botón "Iniciar Experiencia de RA".
  3. Permite el acceso a la cámara.
  4. Escanea los marcadores que se muestran arriba en la sección 2.
